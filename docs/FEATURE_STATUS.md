# 功能完成度清单

状态定义：

- **已实现**：代码链路完整，已有历史构建产物支持，但仍应做目标板回归。
- **部分实现**：主链路存在，仍有标定、容错或产品化工作。
- **模拟**：界面和数据流可演示，数据并非来自真实硬件/网络。
- **未接入**：源码或界面可能存在，但当前主工程没有形成可用功能。

| 模块 | 状态 | 当前实现 | 主要缺口 |
|---|---|---|---|
| STM32U575 BSP | 已实现 | GPIO、I²C1、SPI1、GPDMA、USART1、TIM2、RTC、ICACHE | 需目标板回归 |
| ILI9341 显示 | 已实现 | SPI1 + GPDMA 异步刷新、双 LVGL 绘图缓冲、独立 DMA 缓冲 | 继续压测复位黑屏和长时间刷新 |
| FT6336 触摸 | 已实现/V3修复 | I²C1 读取、横屏坐标转换、LVGL 输入设备 | 需四角、边缘、连续滑动压测 |
| 五页 LVGL UI | 已实现 | Home、Health、Sport、Weather、Call | 资源体积和局部动画仍可优化 |
| 页面滑动 | 已实现 | 统一页面表与左右滑循环 | 需检查快速连续滑动冲突 |
| FreeRTOS 架构 | 已实现 | 4 个任务、消息队列、I²C 互斥锁 | 需测栈水位和长期稳定性 |
| RTC 时间 | 部分实现 | LSI RTC、无效时间时按编译时间初始化 | LSI 精度有限；缺少联网校时和完整掉电策略 |
| SHT20 | 部分实现 | 真实温湿度读取、错误计数 | 失败时自动显示模拟值，UI 未明确标识数据来源 |
| MAX30102 心率 | 部分实现 | 直接驱动、采样任务、波形和 BPM 算法链路 | 需真实硬件、佩戴结构和算法参数验证 |
| MAX30102 血氧 | 实验性 | 比值法估算并限幅 | 未标定，不可作为医疗测量结果 |
| 步数 | 模拟 | 每秒随机增量 | 需接入 IMU/计步算法 |
| 运动模式 | 模拟 | 走路、跑步、骑行状态机与 UI 队列 | 距离、热量、心率均为模型数据 |
| 天气 | 模拟 | 六组本地天气状态轮换 | 需 ESP8266、HTTP/MQTT、JSON 和城市配置 |
| 电量 | 模拟 | 固定显示 86% | 需 ADC/电量计、滤波和电量曲线 |
| 来电页面 | 未接入 | 静态来电卡片和两个按钮 | 按钮无业务回调，缺少通信协议 |
| ESP8266 | 未接入 | `Core/esp8266` 中存在旧版源码 | 未加入当前 Keil 工程业务链路 |
| cJSON | 未使用 | 文件在 Keil 工程中 | 链接结果显示业务未引用 |
| AP3216C | 未接入 | 驱动在工程中 | 当前应用层未使用，且驱动未共享 I²C 互斥锁 |
| 低功耗 | 未完成 | 无完整功耗状态机 | 需背光、传感器、无线和 MCU Stop 模式协同 |
| 背光 PWM | 待核实 | TIM2 CH3 已配置 | 未检索到 `HAL_TIM_PWM_Start`，需结合硬件确认 |
