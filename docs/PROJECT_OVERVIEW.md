# 项目总览

## 1. 项目背景

智能手表需要同时处理图形显示、触摸输入、时间管理、传感器采集和多任务调度，是一个典型的综合嵌入式系统。

本项目使用 STM32U575 作为主控制器，以 FreeRTOS 负责任务调度，以 LVGL 负责图形界面，搭建一套能够持续扩展的智能手表软件平台。

## 2. 项目目标

- 实现 320×240 彩色图形界面；
- 实现电容触摸与页面滑动；
- 建立 FreeRTOS 多任务结构；
- 实现 RTC 时间、日期与表盘；
- 完成温湿度和健康数据采集链路；
- 使用消息队列解耦业务任务与界面；
- 使用互斥锁管理共享 I²C 总线；
- 为联网、计步、电量和低功耗功能预留扩展接口。

## 3. 系统组成

### 主控制器

STM32U575 负责外设控制、任务调度、图形处理、传感器采集和系统通信。

### 显示与触摸

- ILI9341：320×240 TFT 显示屏；
- FT6336：电容触摸控制器；
- LVGL：图形绘制和事件处理；
- SquareLine Studio：页面设计与控件生成。

### 数据采集

- SHT20：环境温湿度；
- MAX30102：心率与血氧原始数据；
- RTC：时间与日期；
- 后续扩展 IMU、电量计与无线通信模块。

## 4. 软件分层

| 层级 | 主要职责 |
|---|---|
| 硬件层 | 提供主控、屏幕、触摸和传感器 |
| 驱动层 | 完成 SPI、DMA、I²C、RTC、TIM 和 UART 访问 |
| FreeRTOS 层 | 完成任务调度、队列通信和资源保护 |
| 应用层 | 完成时间、健康、环境和运动业务 |
| 界面层 | 完成页面展示、动画与触摸交互 |

## 5. 项目特点

- 显示、触摸、RTC 和传感器统一集成；
- LVGL 采用单任务访问模型；
- SPI1 配合 GPDMA 异步刷新；
- I²C 多设备使用互斥锁共享；
- 应用模块相互独立，便于维护；
- 启动过程具有串口日志和阶段状态记录。
